ก่อนเขียนโค้ดต้องดูว่าเกมนี้มีอะไรบ้างก็คือมี field, player, หมวก, และหลุม จากนั้นถึงจะเริ่มคิดว่าจะเก็บข้อมูลแต่ละอย่างยังไง

*Field*
Field คือตาราง 2 มิติ เหมือนตารางช่องสี่เหลี่ยม แต่ละช่องเก็บ character ว่าช่องนั้นเป็นอะไร เช่น ทาง หลุม หรือหมวก การเก็บแบบนี้ทำให้เข้าถึงแต่ละช่องได้ด้วยแถวและคอลัมน์ เหมือน Excel

*Class*
ถ้าไม่ใช้ class ข้อมูลทุกอย่างเช่น field, ตำแหน่ง player, สถานะเกม จะกระจายอยู่เป็นตัวแปรแยกๆ ควบคุมยาก Class ทำให้ข้อมูลและการทำงานอยู่ด้วยกันในที่เดียว ต้องการรู้ตำแหน่ง player ก็แค่ถาม object เดียว

*สร้าง Field แบบ Random ยังไง*
ปัญหาคือจะวาง player, หมวก, และหลุม โดยไม่ให้ทับกันได้ยังไง วิธีที่คิดได้คือรวม position ทุกช่องในตารางเป็น list เดียวก่อน แล้ว shuffle ทั้งหมด หลังจากนั้นหยิบ position แรกให้ player position สองให้หมวก ที่เหลือให้หลุม วิธีนี้การันตีว่าไม่มีทางทับกันได้เลย ไม่ต้องเช็คซ้ำ

*การเดิน เช็คอะไรก่อน*
ทุกครั้งที่กดปุ่ม ต้องเช็คตามลำดับนี้เสมอ อันดับแรกคือออกนอกขอบ field มั้ย ถ้าออกคือแพ้ทันที ต้องเช็คอันนี้ก่อนเพราะถ้าข้ามไปอ่านช่องที่ไม่มีอยู่จริงโปรแกรมจะ crash อันดับสองคือช่องที่เดินไปคือหลุมมั้ย ถ้าใช่คือแพ้ อันดับสามคือช่องนั้นคือหมวกมั้ย ถ้าใช่คือชนะ ถ้าไม่มีอะไรเลยก็เดินต่อได้ปกติ

*Map ดูเหมือนเลื่อนได้*
จริงๆ map ไม่ได้เลื่อน แต่ทุกครั้งที่กดปุ่มโปรแกรมจะล้างหน้าจอแล้ว วาด map ใหม่ทั้งหมด ทำซ้ำเร็วมากจนตาเราเห็นว่า player กำลังเดิน

*รับ Input ยังไง*
ปกติ terminal รอให้กด Enter ก่อนถึงจะส่ง input มาแต่เกมนี้อยากให้กดปุ่มแล้วเลื่อนทันทีโดยไม่ต้องกด Enter เลยต้องบอก Node ให้เปิด raw mode ซึ่งทำให้ทุกปุ่มที่กดถูกส่งมาทันที พอ game over ก็ปิด raw mode แล้วใช้ prompt-sync ถามว่าจะเล่นอีกรอบมั้ย

*ทำไม Constants และ Display แยกกัน*
ข้างในโค้ดเก็บ character จริงๆ เช่น * สำหรับ player แต่ตอนแสดงผลจะแปลงเป็น @ แทน การแยกสองส่วนนี้ออกจากกันทำให้ถ้าอยากเปลี่ยนหน้าตาของ map ก็แค่แก้ที่เดียว ไม่ต้องไปแตะ logic เลย
